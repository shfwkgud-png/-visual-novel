# 낙인의 검 - 비주얼 노벨 개발 인수인계 문서

## 프로젝트 개요
프론티아(Frontia.ai) 대체용 개인 비주얼 노벨 웹앱. 활협전 스타일의 다크 판타지 무협(판타지 일러스트 기반). Gemini 3.5 Flash API(무료 티어) 사용. PC + 모바일(GitHub Pages 호스팅) 양쪽 사용 목표.

## 기술 스택
- 순수 HTML/CSS/JS 단일 파일 (`novel.html`)
- Gemini API 직접 호출 (`gemini-3.5-flash` 모델, `generativelanguage.googleapis.com/v1beta`)
- API 키는 시작 화면에서 입력받아 JS 변수에 저장 (localStorage 미사용 — 추가 검토 필요)
- 배포: GitHub repo `visual-novel` → GitHub Pages 예정

## 폴더 구조
```
visual-novel/
├─ novel.html
├─ background/        # 배경 png 200+개 (alley_*, forest_*, desert_*, coastal_*, mountain_* 등)
└─ character/
   ├─ 25_upper/       # 상반신, {num}_{1~6}.png — 대화용 (예: 106_1.png)
   ├─ 25_full/        # 전신, f_{num}_{1~3}.png — 장면용
   ├─ 25_full_with_shadow/  # 미사용
   └─ 25_pose/        # 전투태세, p_{num}.png
```

## 캐릭터 ID 체계 (101~125, 25명)
파일명은 숫자(101~125), 코드 내 ID는 `f101` 형식. 스프라이트 로드 시 `f` 제거하고 사용.

### 표정 매핑 (upper)
- _1 평상시 / _2 웃음 / _3 분노·찡그림 / _4 눈물 / _5 고통 / _6 고뇌
- emotion 코드: normal=1, smile=2, angry=3, sad=4, pain=5, worry=6

### 자세 매핑 (full)
- _1 평상시 / _2 대화 리액션 / _3 무기 든 자세
- emotion 코드: normal=1, reaction=2, weapon=3

### pose 폴더
- 전투씬 전용, p_{num}.png (표정 변화 없음)

## 세계관 (system prompt에 내장)
**아스테리아 대륙** — 5대 세력 패권 다툼:
- 레온하르트 왕국 (노왕 레오나르, 부패)
- 자유도시 연맹 (항구도시 베른 = 시작 지점)
- 북방 부족 (프레야 고향)
- 교회 국가 (세드릭)
- 마법사 탑 (아르카누스)
- 배후에 비밀결사 "그림자 의회"가 왕국들 조종

**주인공: 카이든** (22세) — 남부 베링 백작가 외아들. 3년 전 그림자 의회 음모로 가문이 역적으로 몰려 멸문. 신분 숨기고 베른 뒷골목에서 막일. 오른손목에 죄인 낙인. 목표: 진상규명 + 복수 + 오명회복. 주인공 스프라이트는 없음(얼굴 안 나옴).

## 동적 캐릭터 주입
모든 캐릭터를 매번 주입하면 컨텍스트 초과 + 할루시네이션 발생. 따라서:
- 세계관 + 주인공 = 항상 고정 주입
- 캐릭터 설정 = 현재 씬 등장 인물만 `getCharPrompt()`로 추가 주입
- `detectActiveChars()`가 최근 히스토리에서 캐릭터 이름 매칭해 등장 인물 추출 (최대 4명)

## 구현 완료 기능
- API 키 입력 화면
- 타이핑 효과 + 탭하여 진행 (스킵 가능)
- 화자 스프라이트 강조 / 나머지 어둡게
- 감정별 표정 자동 전환
- 배경 전환 (setBg)
- 상태창 (평판/골드/위치/신분)
- 선택지 + 자유 입력 동시 지원

## ⚠️ 미해결 / 작업 필요 (우선순위 순)

### 1. 배경 매핑 [최우선 — 현재 검은 화면]
**문제:** AI가 `background` 필드에 파일명을 임의로 생성 → 실제 파일과 불일치 → 검은 화면.
**해결방안:**
- (A) `background/` 폴더 실제 파일명 전부 스캔 → 카테고리별로 정리한 리스트를 system prompt에 주입하고 "이 목록에서만 선택" 강제
- (B) 또는 상황 키워드 → 배경 그룹 매핑 테이블을 코드에 만들고, AI는 그룹명만 반환 → 코드가 그룹 내 랜덤/순차 선택
- 파일명 패턴: 대부분 `{지역}_{장소}{번호}.png` (forest_glade1, desert_oasis2, coastal_tavern3 등). map_*, bg_* 도 있음.
- 권장: (A)로 가되 200개 전체가 아니라 대표 배경 40~60개만 선별해 토큰 절약

### 2. 스프라이트 onerror 처리
현재 이미지 로드 실패 시 `display:none`. 파일명 규칙 안 맞으면 캐릭터가 안 보임. 경로 검증 필요. (upper는 `{num}_{n}.png`, full은 `f_{num}_{n}.png`, pose는 `p_{num}.png` — 접두사 불일치 주의)

### 3. 세이브/로드
현재 새로고침하면 진행 날아감. localStorage로 gameHistory + gameState 저장 필요. (단, API 키 저장은 보안 고려)

### 4. 컨텍스트 관리
`gameHistory.slice(-10)`로 최근 10개만 전송 중. 장기 플레이 시 앞 스토리 망각. 요약(summary) 메커니즘 또는 주요 flag 기반 컨텍스트 유지 필요.

### 5. detectActiveChars 정확도
단순 이름 문자열 매칭이라 "레온"(f113)과 "레오나르"(f107)처럼 부분일치 충돌 가능. 정확한 매칭 로직 개선 필요.

### 6. 모바일 최적화
@media 쿼리 일부 있으나 실기기 테스트 필요. 스프라이트 크기, 입력창 키보드 올라올 때 레이아웃 등.

### 7. API 키 보안
GitHub Pages는 정적 호스팅이라 키가 클라이언트 노출됨. 개인용이라 큰 문제 없지만, 공개 repo면 키 유출 주의. localStorage 저장 + .gitignore 또는 private repo 권장.
